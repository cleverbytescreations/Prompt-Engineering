You are bootstrapping the Claude config for an Android project. The set of
skills, commands, and hooks you create MUST be derived from this repo's
actual stack — do not assume any specific libraries, features, or domain.

# Prime directive
Every artifact you produce is judged on one metric: tokens-per-successful-task.
A smaller, sharper config beats a larger, thorough one. When in doubt, omit.

Three failure modes to avoid above all else:
  1. Blanket file-level denies (e.g. `Read(<file>)`) that block every
     legitimate read of a file because ONE field in it is sensitive.
     Always protect at the FIELD level, never the FILE level.
  2. Recursive greps that wander into generated directories (build/,
     .gradle/, .idea/, out/) and scan 10k–40k files per call. These
     burn 20–90 seconds and return only compiled noise.
  3. Pattern-only signing denies (e.g. `Bash(grep*storePassword*)`) that
     match the grep PATTERN string and block legitimate security-audit
     commands like `grep -r "storePassword" app/src/`. Always scope
     signing denies to the TARGET FILE, not just the field name:
       WRONG: "Bash(grep*storePassword*)"
       RIGHT: "Bash(grep*storePassword*build.gradle.kts*)"
     Cover unscoped searches (grep -r . or git grep without a path)
     in the guard-build-dir-search.sh hook instead.

Rules for the `permissions` block:

PRECEDENCE: In Claude Code, when a command matches BOTH an allow and a deny
pattern, DENY WINS. The whole design below relies on this:
  allow "Bash(grep*build.gradle.kts*)"   ← safe greps pass
  deny  "Bash(grep*storePassword*build.gradle.kts*)"   ← signing greps blocked
Both match `grep storePassword app/build.gradle.kts`, and deny wins so the
extraction is blocked while ordinary build-config greps still pass. Generators
that flatten this into a single rule will break the design.

BASENAME DERIVATION: For each entry in `secrets`, compute:
  <secret-file-basename> = basename of the file path
    e.g. "app/build.gradle.kts"   → "build.gradle.kts"
         "local.properties"        → "local.properties"
         "gradle.properties"       → "gradle.properties"
Use the BASENAME (not the full path) in the deny/allow glob patterns.
Permission rules are glob-matched against the literal command string —
basenames give the widest correct match (covers both `./app/build.gradle.kts`
and absolute paths Claude may use at runtime).

PROTECT SECRETS AT THE FIELD LEVEL, NOT THE FILE LEVEL. A blanket
`Read(<file>)` deny blocks every legitimate read of that file (buildConfig,
SDK levels, dependencies, desugaring flags, variants) and forces every task
that needs build metadata to fail or prompt. Do not do this.

For every entry in the `secrets` Capability Map slot:
- Add the file path to `allow` in BOTH path forms so legitimate reads work:
    "Read(app/build.gradle.kts)"
    "Read(./app/build.gradle.kts)"
- Add a `Bash(grep*<basename>*)` allow so safe grep on the file works.
- For each sensitive field name, add a deny SCOPED TO THE SECRET FILE:
    "Bash(grep*<field-name>*<secret-file-basename>*)"
  NOT:
    "Bash(grep*<field-name>*)"   ← WRONG: blocks audit of source files too
  The glob order matters: field-name comes before file-name so it matches
  `grep storePassword build.gradle.kts` but NOT
  `grep -r storePassword app/src/main/java/` (the latter is a valid
  security audit of new code and must not be blocked).
- Add a narrow `cat` deny tied to the file basename:
    "Bash(cat*<secret-file-basename>*)"
- Enable the `guard-signing-read.sh` Read-tool hook (see scripts section)
  so the model gets a hard reminder of the field names whenever the file
  is read by the Read tool.
- Add a one-line Key Rule in CLAUDE.md citing the file by name.
- The guard-build-dir-search.sh hook covers the remaining gap: unscoped
  `grep -r <field>` or `grep -r <field> .` with no path that could
  still reach the secret file. That hook checks for signing field names
  without a src/-scoped path and blocks with a redirect message.

Allowlist the narrow gradle/adb/git commands actually used in CI or docs,
plus `Bash(git grep*)` and `Bash(grep*libs.versions.toml*)` — these are the
preferred search commands and prompting for them on every search is pure
token waste.

  # guard-gradle-build.sh
  #!/usr/bin/env bash
  set -euo pipefail
  payload="$(cat || true)"
  if command -v jq >/dev/null 2>&1; then
    cmd="$(printf '%s' "$payload" | jq -r '.tool_input.command // .command // empty')"
  else
    cmd="$(printf '%s' "$payload" | sed -n 's/.*"command"[[:space:]]*:[[:space:]]*"\([^"]*\)".*/\1/p' | head -n1)"
  fi
  case "$cmd" in *"gradlew"*) ;; *) exit 0 ;; esac
  case "$cmd" in
    *"assemble"*|*"install"*|*"test"*|*"lint"*|*"dependencies"*|*"--stop"*|*"clean"*) exit 0 ;;
  esac
  if printf '%s' "$cmd" | grep -Eq 'gradlew([[:space:]]+-[^[:space:]]+)*[[:space:]]+(:[a-zA-Z0-9_-]+:)?build([[:space:]]|$)'; then
    {
      echo "Blocked: \`./gradlew build\` triggers a full build."
      echo "Prefer: ./gradlew <observed-narrow-task> --offline"
    } 1>&2
    exit 2
  fi
  exit 0


  # guard-build-dir-search.sh
  #!/usr/bin/env bash
  set -euo pipefail
  payload="$(cat || true)"
  if command -v jq >/dev/null 2>&1; then
    cmd="$(printf '%s' "$payload" | jq -r '.tool_input.command // .command // empty')"
  else
    cmd="$(printf '%s' "$payload" | sed -n 's/.*"command"[[:space:]]*:[[:space:]]*"\([^"]*\)".*/\1/p' | head -n1)"
  fi
  # 1. Hard-block grep/find that target generated directories.
  if printf '%s' "$cmd" | grep -Eq '(grep|find)[[:space:]].*[[:space:]](\.|/)?build(/|$)'; then
    {
      echo "Blocked: searching inside build/ is slow and returns only generated/compiled artifacts."
      echo "Use:  git grep '<pattern>' -- 'app/src/'"
      echo "Or:   find app/build/generated -name '<filename>' 2>/dev/null  (if you specifically need a generated file)"
    } 1>&2
    exit 2
  fi
  # 2. Re-route: grep for BuildConfig → read build.gradle.kts.
  if printf '%s' "$cmd" | grep -Eq "grep.*BuildConfig"; then
    if printf '%s' "$cmd" | grep -Eq "grep.*BuildConfig.*src/"; then exit 0; fi
    {
      echo "Blocked: grepping for BuildConfig searches generated code."
      echo "For VALUES:  Read app/build.gradle.kts directly."
      echo "For USAGES:  git grep 'BuildConfig' -- 'app/src/'"
    } 1>&2
    exit 2
  fi
  # 3. Hard-block grep for signing field names without a src/-scoped path.
  #    Populate the field list at generation time from the `secrets` slot.
  #    Without this, an unscoped `grep -r storePassword .` would bypass the
  #    file-scoped deny rules and still read app/build.gradle.kts.
  signing_fields="storePassword|keyPassword|storeFile|keyAlias|signingConfig"
  if printf '%s' "$cmd" | grep -Eq "grep.*(${signing_fields})"; then
    # Allow if the path is explicitly scoped to source code (security audit).
    # Cover all common source-tree path tokens, not just `src/`.
    if printf '%s' "$cmd" | grep -Eq "src/|app/src|main/java|main/kotlin"; then
      exit 0
    fi
    {
      echo "Blocked: unscoped grep for signing field names could read app/build.gradle.kts."
      echo "To audit NEW source code for signing leakage, scope to src/:"
      echo "  grep -r \"storePassword\\|keyPassword\" app/src/main/java/"
      echo "  git grep 'storePassword\\|keyPassword' -- 'app/src/'"
    } 1>&2
    exit 2
  fi
  # 4. Warn-only on unscoped recursive grep.
  if printf '%s' "$cmd" | grep -Eq "^grep -r|^grep --recursive"; then
    if ! printf '%s' "$cmd" | grep -Eq "(src/|git grep)"; then
      echo "Warning: unscoped recursive grep will scan build/ and .gradle/. Prefer: git grep '<pattern>' -- 'app/src/'" 1>&2
    fi
  fi
  exit 0


  # guard-signing-read.sh
  #!/usr/bin/env bash
  set -euo pipefail
  payload="$(cat || true)"
  if command -v jq >/dev/null 2>&1; then
    file_path="$(printf '%s' "$payload" | jq -r '.tool_input.file_path // .file_path // empty')"
  else
    file_path="$(printf '%s' "$payload" | sed -n 's/.*"file_path"[[:space:]]*:[[:space:]]*"\([^"]*\)".*/\1/p' | head -n1)"
  fi
  # Replace the path glob and field list at generation time from the `secrets` slot.
  case "$file_path" in
    *app/build.gradle.kts) ;;
    *) exit 0 ;;
  esac
  {
    echo "REMINDER: Reading app/build.gradle.kts."
    echo "Do NOT echo, quote, or copy these field values into answers, diffs, or code:"
    echo "  storeFile, storePassword, keyAlias, keyPassword"
    echo "Read this file only for: buildConfig, SDK levels, compileOptions, dependencies, variants."
  } 1>&2
  exit 0


  # suggest-find-feature.sh
  #!/usr/bin/env bash
  set -euo pipefail
  payload="$(cat || true)"

        ;;
    esac
  done
  exit 0

# Step 4 — Token-efficiency rules embedded in every skill

Copy these verbatim into each SKILL.md so they apply when the skill is loaded:
1. Read with `offset`/`limit` for files > 300 lines; never re-read this session
   unless the file was edited.
2. Search with `git grep` first — it respects .gitignore and auto-excludes
   build/, .gradle/, .idea/, out/. Use plain `grep -r` only when you have
   a specific path scope (`-- 'app/src/'`).
3. Never search inside `build/`, `.gradle/`, `.idea/`, `out/` — these
   contain only generated/compiled artifacts. For `BuildConfig.*` values,
   read app/build.gradle.kts directly; do not grep for the generated class.
4. Use Grep with `output_mode:"files_with_matches"` first; targeted reads
   after.
5. Parallelize independent reads/greps in one tool block.
6. For investigations needing > 3 queries, spawn an Explore subagent to
   keep the main context clean.
