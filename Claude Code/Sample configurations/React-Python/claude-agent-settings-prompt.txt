# Prompt: Generate Claude Code Configuration for a New Project

You are setting up Claude Code configuration for a software project from scratch.
Before generating anything, **read the project** to discover its actual structure:

1. Read `README.md` (or root-level docs) — domain, purpose, users
2. List the root directory to find top-level folders
3. Read `package.json` / `pyproject.toml` / `requirements.txt` to identify exact tech stack
4. Identify: frontend framework, backend framework, database(s), cache, queue, search,
   object storage, linter(s), test runner(s), migration tool, i18n library
5. Check if a `Docs/` (or `docs/`) folder exists and list its contents
6. Find the task checklist file (commonly `Docs/tasks.md` or similar)
7. Confirm the absolute path to the backend directory (needed for PostToolUse ruff hook)
8. Confirm the relative path to the frontend directory (needed for ts-diagnostics hook)
9. Determine the **database connection topology** from the architecture/solution-architecture
   document and the infrastructure config (compose/helm/terraform) and the DB settings module:
   is there a connection pooler (e.g. PgBouncer, RDS Proxy, pgcat, Supavisor)? What **pool mode**
   (session / transaction / statement)? What **DB driver** (e.g. asyncpg, psycopg, asyncmy) and
   does it use server-side prepared statements? Does the migration tool connect through the
   pooler or directly to the database?
10. Identify any **reference / master data** the system serves (lookup tables, category/
   config tables, sequence configs, enums-in-DB) exposed via read-heavy `GET` endpoints,
   and whether a **read-through cache helper** already exists (e.g. `core/cache.py`) or a
   cache (Redis/equivalent) is available to add one. Note the env var / setting that controls
   its TTL/eviction window, and which admin/screen write paths must invalidate it.

Then generate every file below. Every detail must come from what you found —
never use placeholder names, never copy content from another project.

## Database & Connection Pooling Rules

[INCLUDE THIS SECTION ONLY IF the project connects through a connection pooler in
 transaction or statement pooling mode (PgBouncer, RDS Proxy, pgcat, Supavisor, …) AND

  MUST configure [Redis]

## File 5: `.claude/hooks/prefer_lsp.py`

Adapt the extensions list to match this project's primary languages.

```python
#!/usr/bin/env python3
import json
import sys

data = json.load(sys.stdin)

tool_input = data.get("tool_input", {}) or {}

candidates = []
for key in ("path", "paths", "glob", "pattern", "query", "include"):
    value = tool_input.get(key)
    if isinstance(value, str):
        candidates.append(value)
    elif isinstance(value, list):
        candidates.extend(str(v) for v in value)

text = " ".join(candidates).lower()

# Adjust extensions to match the project's primary typed languages
lsp_exts = ("[.ext1]", "[.ext2]")   # e.g. (".ts", ".tsx", ".py")

if any(ext in text for ext in lsp_exts):
    print(json.dumps({
        "hookSpecificOutput": {
            "hookEventName": "PreToolUse",
            "permissionDecision": "deny",
            "permissionDecisionReason": (
                "For [language] code navigation, use the LSP tool instead of Grep. "
                "Prefer definitions, references, symbols, hover/type info, diagnostics. "
                "Use Grep only as a fallback when LSP cannot answer."
            )
        }
    }))

sys.exit(0)
```

## File 6: `.claude/hooks/ts-diagnostics.sh` (adapt per language)

Prerequisite: `jq` must be installed (`apt install jq` / `brew install jq`).
The script checks for jq at runtime and exits silently if unavailable.

**For TypeScript projects:**

```bash
#!/bin/bash
# Runs tsc --noEmit after every Edit/Write on a .ts/.tsx file.
# Exits 2 when errors found so Claude is forced to address them.

INPUT=$(cat)

# jq is required to parse tool input — skip silently if unavailable
if ! command -v jq &>/dev/null; then
  exit 0
fi

FILE=$(echo "$INPUT" | jq -r '.tool_input.file_path // empty' 2>/dev/null)

if [[ ! "$FILE" =~ \.(ts|tsx)$ ]]; then
  exit 0
fi

FRONTEND_DIR="./[frontend_dir]"   # relative to project root
if [ ! -d "$FRONTEND_DIR" ]; then
  exit 0
fi

DIAG=$(cd "$FRONTEND_DIR" && npx tsc --noEmit --pretty false 2>&1)
TSC_EXIT=$?

if [ $TSC_EXIT -eq 0 ]; then
  exit 0
fi

echo "TypeScript errors detected after editing: $FILE"
echo ""
echo "$DIAG" | head -50
exit 2
```

**For Python-only projects:** replace tsc with `mypy [module] --ignore-missing-imports`.
**For Go/Rust/Java:** replace with the language's type checker / build tool.
