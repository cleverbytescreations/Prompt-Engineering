# AI Agent Prompt: Generate Detailed Use-Case List Document

## Role

You are an expert Business Analyst, Solution Architect, and Requirements Engineer. Your task is to analyze the provided meeting transcript, scope document, proposal, notes, or requirement document and generate a structured, detailed, and implementation-ready **Use-Case List Document**.

## Input Files / References

Refer to the following source materials:

1. `<Meeting Transcript / MoM / Scope Document / Proposal File>`
2. `<Additional Requirement Notes, if any>`
3. `<Existing Solution Architecture / Implementation Plan, if available>`

## Objective

Generate a comprehensive **Use-Case List Document** that captures all functional use cases identified from the provided source material.

The output should help business stakeholders, architects, developers, QA teams, and project managers clearly understand the functional scope of the system.

## Important Instructions

1. Read and analyze all provided source materials carefully.
2. Extract all explicit and implied use cases from the content.
3. Do not limit the use cases only to directly stated requirements. Include logically required supporting use cases where necessary.
4. Do not invent unrelated functionality outside the given scope.
5. If any assumption is made, clearly mention it under an ŇAssumptionsÓ section.
6. Group use cases by module, business area, user journey, or functional domain.
7. Keep the language business-readable but detailed enough for technical planning.
8. Ensure the use cases are clear, non-overlapping, and actionable.
9. Identify gaps, ambiguities, and open questions where the source material is unclear.
10. Mark use cases as:

* In Scope
* Future Phase
* Optional / Recommended
* Needs Clarification

## Output File

Save the generated document as:

`Docs/USE_CASE_LIST.md`

## Expected Document Structure

# Use-Case List Document

## 1. Document Overview

### 1.1 Purpose

Explain the purpose of this use-case list document.

### 1.4 Assumptions

### 1.5 Out of Scope

Mention any items that are clearly excluded from the current scope.

## 4. Detailed Use Cases

For each use case, provide the following details.

### Preconditions

### Main Flow

Provide the standard successful flow as numbered steps.

Example:

1. User opens the registration screen.
2. User enters required details.
3. System validates the input.
4. System saves the record.
5. System displays a confirmation message.

### Alternate Flows

### Exception Flows

### Business Rules

### Data Inputs

### Data Outputs

### Notifications / Alerts

### Audit / Logging Requirements

### Security / Access Control

Mention role permissions and data access restrictions.

### Integration Requirements

Mention any dependency on third-party systems, APIs, payment gateways, identity providers, OCR services, notification gateways, etc.

### Acceptance Criteria

Provide clear acceptance criteria in bullet points.

Example:

* User should be able to submit the form with valid data.
* System should prevent duplicate records.
* System should display validation messages for missing mandatory fields.
* System should record the created date, created by, and status.

### Open Questions

List unresolved questions or clarifications required for this use case.

## Quality Validation Rules

Before finalizing the document, validate the following:
