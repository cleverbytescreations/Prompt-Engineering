Estimation Methodology

Each task checklist entry was reviewed for implementation depth, acceptance criteria, and stated implementation notes. Tasks were grouped into effort buckets:

MediumAuth flow, cursor pagination, moderation action, typed API client1.5 – 3

Not included: Product discovery / UX design time, QA regression cycles beyond automated tests, stakeholder review rounds, external vendor integration SLAs, DevOps/infrastructure provisioning for cloud accounts already configured, and project management overhead (> 15 %).
Phase 1 — MVP

Scope

Full-stack MVP: FastAPI + PostgreSQL + Redis + OpenSearch + MinIO + Celery backend with 15 API modules (Auth, Users, Orgs, Invites, Repository, Q&A, News, Feed, Moderation, Notifications, Search, Dashboard, Taxonomy, Admin, System); transactional outbox; hybrid BM25+k-NN search; embedding pipeline; Next.js 16 App Router frontend with all corresponding views, MSW demo mode, role-based access, and a typed API client.

Team-Size Based Calendar Estimates
Calendar time = combined person-days ÷ (team-size × parallelism efficiency × 22 working days/month). Parallelism efficiency: 0.75 for a 2-person team (high serialization dependency), 0.70 for larger teams (coordination overhead). Includes a 20 % buffer for review cycles and unplanned work.

Key Estimation Risk Factors
Risk Phase(s) Likelihood Impact Mitigation
LangGraph RAG performance — p95 ≤ 2,500 ms budget with 9-node pipeline and live translation P2 High +15–25 days Prototype BACKTRANS node first; accept cached-only warm-path initially
OpenSearch hybrid search tuning — BM25 + kNN RRF weights require iterative calibration P1 Medium +5–10 days Use seeded fixture dataset; define recall@10 baseline before tuning
Neo4j entity extraction accuracy — SpaCy/LLM NER quality on legal text may require fine-tuning P3 High +10–20 days Scope Phase 3 extraction to rule-based patterns first; ML fine-tune is a follow-on
React Native mobile app — Expo + FCM/APNS store submission involves external review latency P3 Medium +8–15 days Submit TestFlight / Play Internal Track early; development proceeds in parallel
Microservices extraction — service boundary cut with shared DB + Redis introduces deployment risk P3 Medium +10–20 days Keep monolith flag in place; canary route 5 % of traffic first
LiveKit self-hosted setup — egress recording to S3 + webhook signature requires infrastructure time P3 Medium +4–8 days Use LiveKit Cloud for Phase 3 GA; self-host only if data-residency is mandated
Locale-prefix route change in Phase 2 — Adding /en /es /fr prefixes touches auth middleware, deep links, and all Playwright tests P2 Medium +4–7 days Plan URL strategy in Phase 1; use next-intl from day one even if only EN is active
Outbox poller concurrency — double-dispatch under concurrent pollers requires careful locking P1 Low +2–4 days SELECT … FOR UPDATE SKIP LOCKED is standard pattern; test early with two pollers
Moderation SLA compliance — 48-hour SLA requires alert + escalation wiring that goes beyond the checklist P1 Low +1–3 days SLO alert in T-L6.8 covers detection; UI SLA countdown is a Phase 1 stretch
Assumptions & Constraints
Included in estimates
  • Developer self-testing, unit + integration test authorship, and code review time (~25 % of implementation time)
  • Sprint ceremonies and stand-ups (~10 % overhead, already embedded in daily rates)
  • Docker Compose local dev environment setup and CI/CD pipeline configuration
  • Basic AWS infrastructure provisioning (RDS, ElastiCache, OpenSearch Service, S3, ECS/EKS task definitions) where referenced in checklists
  • MSW handler authorship for all Phase 1 endpoints (the largest single frontend investment)
NOT included in estimates
  • UX/Product design and wire-framing
  • Manual QA regression cycles and UAT facilitation with stakeholders
  • Legal/compliance review and sign-off turnaround time
  • Third-party API onboarding delays (OpenAI, LiveKit, AWS Translate, FCM/APNS certificates)
  • Project management, scrum-master, and business analysis time
  • Data migration from any existing legacy system
  • Load-testing infrastructure provisioning beyond what is defined in the checklists
